# Contributing to BMP Parser

Thank you for helping improve this BMP parser project.

## Recommended workflow

1. Open an issue first for new features or bug fixes.
2. Fork the repository.
3. Create a branch with a clear name, for example:
   - `feature/bmp-24bit-support`
   - `fix/header-validation`
   - `docs/readme`
4. Make your changes and keep commits focused and descriptive.
5. Submit a pull request with a clear summary of what changed.

## Testing

- Run the demo with `odin run main.odin`.
- Add regression tests to `tests/bmp/bmp_test.odin`.
- Document any new behavior in `README.md` or relevant docs.

## Code style

- Follow existing Odin style and repository conventions.
- Keep functions small and clearly commented.
- Use the existing comment style and language conventions in the repository.

## What to include in a PR

- A concise description of the change.
- What problem it fixes or what feature it adds.
- Any relevant tests added or updated.
- Notes about known limitations or follow-up work.
